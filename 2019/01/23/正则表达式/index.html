发表于 2019-01-23 | 分类于 js基础

正则表达式

描述了一种字符串匹配的模式，可以用来检查一个字符串是否含有某种子串、将匹配的子串做替换或者从某个字符串中取出符合某个条件的子串等。

这个说法挺拗口的，就直接给个例子来说明吧。

这个是通过正则表达式来匹配字符串里面的所有连续的数字，而也是正则表达式的作用。

正则表达式的组成

表达式

[abc] 这个是或的意思，只匹配一个字符 就可以匹配[]里面的任意一个元素，相当于是一个或的集合吧

[0-9] [a-Z][A-z] 同上，分别是 0到9，a到z以及A到Z的意思

而如果需要表示15-80的意思的话，需要这么写

元字符

\d : 0-9之间的任意一个数字 \d只占一个位置

\w : 数字，字母 ，下划线 0-9 a-z A-Z _

\s : 空格或者空白等

\D : 除了\d

\W : 除了\w

\S : 除了\s

. : 除了\n之外的任意一个字符

\ : 转义字符

| : 或者

() : 分组

\n : 匹配换行符

\b : 匹配边界 字符串的开头和结尾 空格的两边都是边界 => 不占用字符串位数

^ : 限定开始位置 => 本身不占位置

$ : 限定结束位置 => 本身不占位置

[a-z] : 任意字母 []中的表示任意一个都可以

[^a-z] : 非字母 []中^代表除了

[abc] : abc三个字母中的任何一个

[^abc]除了这三个字母中的任何一个字符

量词

  • : 0到多个
  • : 1到多个
    ? : 0次或1次 可有可无
    {n} : 正好n次；
    {n,} : n到多次
    {n,m} : n次到m次

正则的函数方法

  1. reg.test(str) 检测里面是否有匹配到正则 返回的是 true和false

  2. reg.exec(str)

关于exec有两个值得注意的地方，

第一点，如果reg的全局匹配的话（g），那么多次调用的时候，返回的内容也不一样，然后会在返回null之后重新开始

第二点，exec返回的内容跟括号()有关系，因此reg有三个括号（），所以返回的是数组有四个值，然后看对应的是哪个()的值就对应到哪个括号。而且他不是看reg来匹配，而是依据括号里面的内容

  1. str.match(reg)

如果不是全局匹配，那么返回的结果跟exec是一摸一样的。另外还有一点需要说明的是，对于返回的数组，真正的值只有四个，其他的index、input、group都只是属性，跟length是一样的

而如果全局匹配的话，返回的值就完全不一样了

一般这个函数在日常编程中经常被用到，比如jQuery的源码中，就有大量使用正则的match方法来进行筛选，在一开始的初始化函数中

正则表达式的特性

  1. 贪婪性

如果符合要求就一直往后匹配，一直到无法匹配为止，这就是贪婪模式。

这个输出的是*,跟前面不一样的就是加了个 + 号，+号的意思就是匹配1到多个的意思，而根据正则表达式的贪婪性，他就会匹配尽可能多的字符，所以就全部匹配到了

  1. 懒惰性
    懒惰性则是正则在成功捕获一次后不管后边的字符串有没有符合条件的都不再捕获。
    如果想捕获目标中所有符合条件的字符串的话，我们可以用标识符g来标明是全局捕获
